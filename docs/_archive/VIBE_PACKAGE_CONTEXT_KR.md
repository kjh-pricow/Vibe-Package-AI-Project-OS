# VIBE_PACKAGE_CONTEXT.md
# Vibe Coding Package 구축을 위한 Source of Truth

---

## 1. 이 문서의 목적

이 문서는 Vibe Coding Package의 배경, 의도, 설계 철학을
**repository 수준의 지침**으로 완전히 이전하기 위해 존재한다.

이 repository에서 작업하는 모든 AI는
이 문서를 **최상위 기준(Source of Truth)** 으로 삼아야 한다.

이 문서를 읽은 AI는:
- 이전 대화 기록을 알 필요가 없어야 하며
- 사용자와 동일한 문제의식을 공유해야 하고
- 맥락 누락 없이 사용자와 협업할 수 있어야 한다

---

## 2. 문제의식 (왜 이 패키지가 필요한가)

### 2.1 기존 vibe 코딩의 한계

일반적인 “vibe 코딩” 방식은 다음에 초점을 둔다:
- 빠르고 즉흥적인 코드 생성
- 대화 흐름 중심의 단기 작업

그 결과:
- 맥락이 사라지고
- 동일한 논의가 반복되며
- 토큰 사용량이 통제되지 않고
- 재현이 불가능하며
- 팀 단위 공유가 어렵고
- CLI나 자동화로 확장할 수 없다

### 2.2 기존 AI 코딩 도구의 한계

대부분의 AI 코딩 도구는:
- IDE 중심이며
- 단일 에이전트 구조이고
- 코드 생성 보조에 초점을 둔다

이들은 다음이 부족하다:
- 프로젝트 단위 사고
- 의사결정 이력 관리
- 지속적인 기억
- 재현 가능한 작업 흐름

**결론:**  
문제는 AI의 성능이 아니라,  
**AI가 일할 수 있는 구조의 부재**다.

---

## 3. Vibe Coding의 재정의

기존 정의:
- vibe 코딩 = 빠른 코드 생성

이 프로젝트에서의 정의:
- vibe 코딩 = 사용자가 자연어로 사고하고 질문하면,
  AI가 그 사고를 구조화·결정·구현으로 변환하는 과정

핵심 중심은:
- Q&A
- 사고 정리
- 명시적인 의사결정
- 구조적 명확성

코드는 중심이 아니라 **결과물**이다.

---

## 4. 핵심 개념: AI를 “팀”으로 다룬다

AI는 단일 전능 도구가 아니라,
**역할을 가진 팀**으로 다뤄진다.

### 개념적 역할

- User: Owner / PM / 최종 의사결정자
- Chief AI: 전체 조율 및 오케스트레이션
- Analyst AI: Q&A를 명세로 구조화
- Dev AI: 명세를 구현으로 변환
- QA AI: 리스크 및 품질 검토
- Librarian AI: 모든 맥락을 파일로 고정

중요한 전제:
- AI는 기억을 신뢰할 수 없다
- 모든 기억은 파일 시스템에 외부화되어야 한다

---

## 5. Vibe Package란 무엇인가

Vibe Package는:
- 단일 대화가 아니며
- 임시 세팅이 아니라
- **재사용 가능한 repository 기반 프로젝트 운영 시스템**이다

### 핵심 원칙

1 Vibe Package = 1 Workspace  
1 Vibe Package = 1 Repository  

이유:
- 맥락 오염 방지
- Git 이력이 곧 의사결정 이력
- CLI 실행 가능
- 환경에 종속되지 않음

---

## 6. Repository 구조 개념

repository-root/
├─ vibe/ # AI 팀, 규칙, 프롬프트, 기억, CLI (엔진)
└─ workspace/ # vibe 코딩으로 생성된 산출물

### vibe/
- AI의 행동 방식을 정의
- 프롬프트, 정책, 메모리 템플릿 포함
- 향후 CLI/런타임 로직 위치

### workspace/
- 코드, PoC, 산출물 저장
- vibe 코딩의 출력 결과

---

## 7. 절대 위반 불가 설계 원칙

1. Source of Truth는 항상 repository 파일이다
2. Chief AI는 패키지 내부에 존재해야 한다 (플랫폼 설정 금지)
3. ChatGPT Project 설정은 개발 보조 수단일 뿐이다
4. 시스템은 향후 CLI 실행이 가능해야 한다
5. 불명확함은 가정이 아니라 질문으로 해결한다
6. 구현보다 구조가 항상 우선이다

---

## 8. 이 프로젝트의 정체성

이 프로젝트는 다음이 아니다:
- “AI로 더 빨리 코딩하기”
- “프롬프트 트릭”
- “ChatGPT 전용 워크플로우”

이 프로젝트는 다음이다:
- AI가 일하는 방식을 repository 수준에서 정의하는 것
- AI 협업을 구조적으로 가능하게 하는 시스템
- 재현 가능하고, 검증 가능하며, 이식 가능한 AI 운영 모델

---

## 9. 이 문서를 읽은 AI에게 기대되는 태도

이 문서를 읽은 AI는 반드시:
- 사용자의 원래 문제의식을 공유해야 하며
- 왜 속도보다 구조를 우선하는지 이해해야 하고
- 성급하게 구현으로 들어가지 말아야 하며
- 항상 “지금 우리가 정의하고 있는 것은 무엇인가?”를 기준으로 행동해야 한다

이 repository는 현재 **정의 단계**에 있다.

- 규칙
